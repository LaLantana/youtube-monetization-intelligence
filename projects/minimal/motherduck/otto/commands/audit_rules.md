# Audit Rules Command

Review all project rules in `otto/rules/` against the quality standards in `learning.md`.

## Steps

1. Read `otto/rules/learning.md` to understand the quality standards
2. For each rule file in `otto/rules/`:
   - Check if it's **Clear**: Unambiguous, actionable guidance
   - Check if it's **Concise**: Minimal words to convey the point
   - Check if it's **Non-duplicative**: Doesn't repeat `ascend.*` rules
   - Check if it's **Non-conflicting**: Doesn't contradict other rules
3. Report findings with specific recommendations for each rule that doesn't meet standards
