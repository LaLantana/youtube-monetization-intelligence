# README Maintenance

Keep project READMEs accurate and up-to-date as you work.

## When Working on Components or Flows

1. Read the relevant README first to understand patterns
2. Update READMEs proactively after changes (new components, pattern changes, config updates)
3. Keep cross-references accurate when files move

## What to Update

- Component lists in flow READMEs
- Code examples to match current patterns
- File links using `@path` syntax
- Business logic descriptions
- New patterns worth documenting

## Style Guidelines

- Match existing format and structure
- Use proper smart link syntax: `@path`, `#component:name`, `#flow:name`, `#connection:name`, `#automation:name`
- Keep content scannable with headings and lists
- Update cross-references when changing related READMEs
